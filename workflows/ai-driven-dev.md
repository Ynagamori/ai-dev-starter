# AI 駆動開発の基本ワークフロー

このドキュメントは、ChatGPT / Claude / Gemini などを使って
「AI 主体で開発を進める」ための基本フローを定義します。

## 0. プロジェクトのコンテキスト共有

1. プロジェクトの目的・ユーザー・主要機能を箇条書きでまとめる
2. それを AI に渡し、「理解した内容を要約して」と依頼する
3. 認識にズレがあれば、このタイミングで修正する

## 1. 機能単位のタスク分解

1. 実装したい機能を 1 つ選ぶ（例: 「イベント作成フォーム」）
2. AI に「この機能を実装するためのタスクを 5〜7 個に分解して」と依頼
3. タスクリストをそのまま Issue として起票する

## 2. 設計フェーズ

1. 画面 / API / DB のどこから手を付けるかを AI と相談して決める
2. 必要に応じて「ディレクトリ構成案」「コンポーネント構成案」を AI に出してもらう
3. 気になる点があれば、その場で質問・修正する

## 3. 実装フェーズ

1. タスクごとに、AI に「このタスクだけを実装して」と依頼する
2. AI の出したコードを手元で貼り付け・実行し、エラーが出たらそのログを AI に渡す
3. 1 タスクが終わったら、Git でコミットする（コミットメッセージも AI に提案させてもよい）

## 4. レビュー & リファクタリング

1. ある程度実装が進んだら、AI に「このファイルの改善点をレビューして」と依頼する
2. パフォーマンス・可読性・保守性など、観点を指定するとより有用な指摘を得られる
3. 反映するものだけを選び、必要に応じて再度レビューを依頼する

## 5. ドキュメント生成

1. 最終的な構成が固まったら、AI に README やアーキテクチャ説明を書いてもらう
2. 「このディレクトリ構成と主要ファイルを説明する README を書いて」と指示する
3. 生成されたドキュメントを、人間の視点で最終調整する

---

このワークフロー自体も、実際に使いながら改善していくことを前提としています。
改善案があれば Issue / PR を歓迎します。
